A baisc understanding of Angular Js

Angular Js is a javascripts framework and it is developed and maintained by Google. You can inlcude angular Js using html scripts tag as

<script src="http://ajax.googleapis.com/ajax/libs/angularjs/1.4.8/angular.min.js"></script>
You can get the Angular Js library from global repository or you can download it to your project directory and then you can include it.

Angular Js is based on MVC patterns and hence it seperates the application into Views,Models and Controllers.
Angular Js Life Cycle:-
It is very important to know about life cycle of any application.Here I will try to explain the life cycle of Angular Js. A good understanding of application life cycle always helps in Devloping a robust Application.
Usually, Angular Js application has three life cycle: The three phases occurs each time a web is loaded in the browser.
  1. Bootstrap Phase or Initialisation Phase
  2. Compilation Phase
  3. Runtime Data Binding Phase
Bootstrap or Initialisation Phase:
The first phase of Angular Js application is bootstrap phase.This phase started as soon as you open your application in browser and Angular js library is downloaded to the browser.Once Angular Js library is downloaded, Angular initalises it's own necessary components.Now it's time to initialise your application module which you register through ng-app dirctive(Don't worry with new terms ng-app, You will get it soon in next chapter). Through ng-app your application module will be loaded and also any dependencies are injected.
Compilation Phase:
The second phase of the Angular js life cycle is HTML compilation.Initally when a web page is loaded in the browser, a static form of the DOM is loaded in the browser.During the Angular Js compilation phase,the static DOM is replaced with dynamic DOM wich is nothing but the Angular Js View.
This phase actually inlcudes traversing the entire static DOM to find all Angular Js directive and Custom directive. Once it finds all the directive,it links the Angular Js directive to Javascripts code in Angular Library and custom directive to Javascripts code available your application.
Runtime Data Binding Phase:
The final phase of the AngularJS application is the runtime phase, which exists until the user reloads or navigates away from a web page. At that point, any changes in the scope are reflected in the view, and any changes in the view are directly updated in the scope, making the scope the single source of data for the view.
AngularJS behaves differently from traditional methods of binding data. Traditional methods combine a template with data received from the engine and then manipulate the DOM each time the data changes. AngularJS compiles the DOM only once and then links the compiled template as necessary, making it much more efficient than traditional methods.

A Simple Hello World Example:- I will try to explain MVC architecture in Angular Js uisng a very simple Hello World example. Let's do the coding 
<!DOCTYPE html>
<html lang="en">
<head>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.2.5/angular.min.js"></script>
</head>
<body ng-app="myapp">
 <div ng-controller="HelloController" >
  <h2>Hello {{helloTo.title}} !</h2>
 </div>
 <script>
  var myApp=angular.module("myapp", [])
      myApp.controller("HelloController", function($scope) {
           $scope.helloTo = {};
           $scope.helloTo.title = "World, AngularJS";
        } );
 </script>
</body>
</html>

  1. An angular js app start with defining the application name and this is done with ng-app="anyAppNameYouWish" (in above exapmle it is ng-app="myapp"). You can have more than one application name,it depends up to you how you design your application.
    You can include ng-app(an angular built-in directive) to any html element(like html,body or div).But I suggest to include it in html or body tag,this ensure that entire page belongs to that application. In Hellow World example,entire page belongs to "myapp".
  2. Now you are going to declare your controller using another anular built-in directive ng-controller.In hellow world example,it has delcared as ng-controller="HelloController". Always declare your controller with a logical name(depending on it's usage).
  3. Now your view part is ready. It's time to register your controller. To register your controller with application "myapp",you need a reference to "myapp". You can get the reference with var myApp=angular.module("myapp", []). Now myApp has refrence to "myapp" and you can register your controller using the code
    myApp.controller("HelloController", function($scope) {
    $scope.helloTo = {};
    $scope.helloTo.title = "World, AngularJS";
    } );
